# PRD — 솔로몬스터디카페 관리 시스템

**버전**: 0.1.0  
**최종 수정**: 2026-06-26  
**대상 독자**: 개발자, 운영진

---

## 1. 제품 개요

솔로몬스터디카페는 학생들이 예약 없이 교시 단위로 자리를 사용하는 스터디카페입니다. 이 시스템은 기존에 카카오톡·종이 장부로 운영하던 출결 관리, 외출·파워냅 신청, 벌점 기록을 디지털화하여 운영 효율을 높이는 것을 목표로 합니다.

**핵심 사용자**
- 학생 (수십~수백 명): 스마트폰 모바일 위주 사용
- 관리자 (1~3명): 데스크톱 위주 사용

---

## 2. 핵심 요구사항

### 2.1 교시 시스템

- 1일 8교시 운영 (기본값: 09:00~20:50, 10분 휴식 포함)
- 교시 시간은 관리자가 `periods` 테이블을 통해 변경 가능
- 현재 교시, 다음 교시를 실시간으로 계산하여 표시
- 요일(월~일)×교시(1~8) 격자로 주간 시간표 관리

### 2.2 시간표 시스템

- 학생은 매주 월요일 기준으로 주간 시간표를 제출
- 제출된 시간표는 `draft` → `submitted` 상태로 변경
- 관리자는 학생의 시간표를 조회·수정 가능
- 제출 기한 잠금/해제 기능 (`schedule_unlock` 설정)
- 시간표 수정 이력은 `activity_logs`에 기록

### 2.3 출결 시스템

**출석 상태 종류**
| 상태 | 설명 |
|---|---|
| `present` | 출석 |
| `absent` | 결석 |
| `late` | 지각 |
| `early_leave` | 조퇴 |
| `excused_absence` | 공결 (승인된 결석 신청) |
| `excused_early_leave` | 공조퇴 (승인된 조퇴 신청) |

- 자동 처리: 교시 시작 시각 기준으로 무단결석 자동 감지 (DB 트리거)
- 수동 처리: 관리자가 개별 출석 상태 변경 가능
- 학생 출처(`source`): `self`(학생 자가 체크), `admin`(관리자), `system`(자동)

### 2.4 신청 시스템

- **결석 신청**: 사전 결석 신청 → 관리자 승인 시 `excused_absence`로 자동 처리
- **조퇴 신청**: 사전 조퇴 신청 → 관리자 승인 시 `excused_early_leave`로 자동 처리
- 신청 상태: `pending` → `approved` / `rejected`
- 반려 시 사유 입력 필수
- 처리 이력은 `request_logs` 테이블에 기록

### 2.5 외출 시스템

- 학생이 앱에서 외출 시작 버튼 클릭 → `bathroom_logs` INSERT
- 복귀 시 복귀 버튼 클릭 → `status = 'completed'`로 UPDATE
- 학생 1명당 동시에 1건의 외출만 허용 (DB 유니크 제약)
- 미복귀 시 `overdue` 상태로 변경 및 벌점 자동 부여 가능
- 관리자 대시보드에 실시간으로 외출 인원 표시

### 2.6 파워냅 시스템

- 1일 1회, 최대 40분 제한 (`POWER_NAP_MAX_MINUTES = 40`)
- 학생이 파워냅 시작 → `planned_end_at` 자동 계산
- 시간 초과 시 `overdue` 상태, 무단 파워냅 시 `unauthorized` 및 벌점 5점
- `student_id + nap_date` 유니크 제약으로 중복 방지

### 2.7 벌점·경고 시스템

**벌점 규칙** (일부)
| 사유 | 점수 |
|---|---|
| 지각, 외출 복귀 지연, 졸음 등 경미한 위반 | 2점 |
| 무단 조퇴, 전자기기 사용 등 중간 위반 | 5점 |
| 무단 결석, 관리자 지시 불이행 등 심각한 위반 | 10점 |

**경고 임계값**
| 누적 벌점 | 조치 |
|---|---|
| 10점 | 1차 경고 |
| 20점 | 2차 경고 |
| 30점 | 퇴원 처분 |

- 경고 발생 시 DB 트리거가 `warning_records` INSERT + 알림 생성
- 관리자가 수동으로 벌점 부여/차감 가능
- 경고 후 벌점 초기화 여부는 관리자 재량

### 2.8 알림 시스템

- 알림 대상: 관리자 (학생 대상 알림 현재 미구현)
- 트리거 이벤트: 외출 시작/복귀, 파워냅 시작/종료, 경고 발생, 무단결석 감지
- 알림은 `notifications` 테이블에 저장되며 Supabase Realtime으로 즉시 전달
- 읽음 처리 가능

### 2.9 1:1 채팅 시스템

- 학생당 채팅방 1개 (학생 ↔ 관리자 전용)
- Supabase Realtime으로 실시간 메시지 전달
- 기능: 텍스트 메시지, 이미지 첨부 (드래그&드롭), 읽음 확인, 입력 중 표시
- 관리자 채팅 화면: 좌측 학생 목록 + 중앙 채팅 + 우측 학생 상태 패널
- 빠른 답변(Quick Reply) 템플릿 관리
- 메시지는 최근 100건 표시 (추가 페이지네이션 미구현)
- 이미지 업로드: Supabase Storage `chat-images` 버킷 (5MB, image/* 제한)

### 2.10 학생 상태 패널 (관리자용)

관리자 채팅 우측 패널에서 선택된 학생의 정보를 실시간으로 조회:
- 기본정보 (이름, 전화번호, 학교, 학년, 등록일)
- 이용권 상태
- 현재 교시 정보
- 오늘 공부 현황 (출석/외출/파워냅)
- 벌점 및 경고 수
- 오늘 시간표
- 최근 신청 내역

퀵 액션: 벌점 부여, 경고 부여, 출석 처리, 외출 종료, 파워냅 종료, 공지 보내기

### 2.11 실시간 관제 (관리자용)

관리자가 스터디카페 전체를 "관제판"처럼 한눈에 내려다보는 화면 (`/admin/monitor`).

**좌석 배치 중심 뷰**
- 실제 책상 배치를 그대로 표현 (현재 8석, 4×2 + 통로)
- 좌석 위치는 `seat_layouts` 테이블로 관리 (하드코딩 금지, 향후 드래그앤드롭 편집 대비)
- 반응형: 모바일 2열 / 태블릿·PC 4열 + 통로 구분선
- 각 좌석 카드 표시: 좌석번호, 학생 이름, 현재 상태, 현재 교시, 남은 시간

**좌석 상태 색상** (카드 전체 색상으로 즉시 식별)
- 🟢 공부중 / 🚻 외출중 / 😴 파워냅 / 🔴 지각 / ⚫ 미등원·결석·비활성 / ⚪ 빈 좌석

**좌석 클릭 시**
- 배정된 좌석: 우측에 학생 Live Status Panel + 채팅창 동시 표시 (페이지 이동 없음)
- 빈 좌석: 학생 배정 모달 (활성 회원 검색 후 선택 → 배정)
- 모바일에서는 하단 시트(bottom sheet) 형태로 표시

**좌석 배정 관리 (관리자 웹)**
- 빈 좌석 클릭 → 학생 배정 (이미 다른 좌석을 쓰던 학생을 선택하면 자동 이동, 1인 1석)
- 선택된 학생 패널 상단에서 "좌석 해제"
- `student_profiles.seat_number` UPDATE로 처리, Realtime으로 즉시 반영

**실시간 이벤트 로그**
- 우측 하단에 외출 시작/복귀, 파워냅 시작/종료, 벌점 부여 이벤트를 시간순 표시
- Supabase Realtime으로 자동 추가 (폴링 없음)

**상단 요약**
- 착석/외출/파워냅/결석/지각/전체 인원을 작은 Summary Card로 배치 (메인은 좌석 배치)
- 현재 교시 및 잔여 시간, 실시간 시계 표시

**향후 확장 설계**
- CCTV AI가 학생 상태(공부/휴대폰/졸음/이탈 등)를 자동 변경하면 화면이 즉시 반영되도록 구조화
- 상단에 지점 선택 추가가 쉽도록 단일 지점 기준으로 설계

## 3. 인증 방식

- **학생**: 전화번호 + 비밀번호 (이메일 변환 후 Supabase Auth 사용: `phone@solomonstudy.app`)
- **관리자**: 전화번호 + 비밀번호 (동일 방식)
- **구글 OAuth**: 학생 전용 (구글 로그인 후 `/auth/register`에서 이름·전화번호 등록)
- 세션 자동 갱신 (`autoRefreshToken: true`)

---

## 4. 비기능 요구사항

- **모바일 최우선**: 학생 화면은 iOS/Android 브라우저에서 사용 가능해야 함
- **PWA**: 홈 화면 추가 지원 (오프라인은 미지원)
- **실시간성**: 관리자 대시보드와 채팅은 새로고침 없이 즉시 반영
- **보안**: RLS로 학생이 타 학생 데이터에 접근 불가
- **폴링 없음**: 모든 실시간 데이터는 Supabase Realtime Websocket으로 처리
